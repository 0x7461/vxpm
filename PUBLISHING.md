# Publishing vpm to GitHub

Steps to go from local-only → public repo → xbps-src template.

---

## Step 1: Create GitHub repo

```sh
gh repo create 0x7461/vpm --public --description "TUI for managing custom void-packages templates" --source . --push
```

This creates the repo, sets origin, and pushes all commits in one shot.

---

## Step 2: Create a release

Tag and push:
```sh
git tag v0.1.0
git push origin main --tags
```

Then create the GitHub release:
```sh
gh release create v0.1.0 --title "v0.1.0 — Initial release" --notes "First public release."
```

The source tarball is auto-generated by GitHub at:
`https://github.com/0x7461/vpm/archive/v0.1.0.tar.gz`

---

## Step 3: Compute checksum for xbps-src template

```sh
curl -sL https://github.com/0x7461/vpm/archive/v0.1.0.tar.gz | sha256sum
```

Update `~/void-packages/srcpkgs/voidpm/template`:
- Replace `PLACEHOLDER` in `homepage`, `changelog`, `distfiles` with `https://github.com/0x7461/vpm`
- Replace `checksum=PLACEHOLDER` with real sha256

---

## Step 4: GitHub Actions CI (optional but good)

File: `.github/workflows/release.yml`

Builds a Linux x86_64 binary on tag push.
Uploads binary to the release as an asset.
Enables the xbps-src template to use a pre-built binary instead of building from source.

See below.

---

## Step 5: Build and install via vpm itself

```sh
cd ~/void-packages
./xbps-src pkg voidpm
xi voidpm
```

---

## GitHub Actions release workflow

`.github/workflows/release.yml`:

```yaml
name: Release

on:
  push:
    tags:
      - 'v*'

jobs:
  build:
    name: Build Linux x86_64
    runs-on: ubuntu-latest

    steps:
      - uses: actions/checkout@v4

      - name: Install Rust
        uses: dtolnay/rust-toolchain@stable

      - name: Cache cargo
        uses: actions/cache@v4
        with:
          path: |
            ~/.cargo/registry
            ~/.cargo/git
            target/
          key: ${{ runner.os }}-cargo-${{ hashFiles('**/Cargo.lock') }}

      - name: Build release binary
        run: cargo build --release

      - name: Create archive
        run: |
          mkdir -p dist
          cp target/release/vpm dist/
          cp LICENSE README.md dist/
          tar -czf vpm-linux-x86_64.tar.gz -C dist .

      - name: Upload to release
        uses: softprops/action-gh-release@v2
        with:
          files: vpm-linux-x86_64.tar.gz
```

With this in place, the xbps-src template can use the binary repack pattern instead of building from source:

```sh
# Simplified template using pre-built binary:
distfiles="https://github.com/0x7461/vpm/releases/download/v${version}/vpm-linux-x86_64.tar.gz"
# No cargo build needed — use do_install directly
```

This avoids the cargo build step in xbps-src (which needs internet access) and makes the package much faster to install.

---

## Name note

Package name in void-packages: `voidpm` (since `vpm` is already taken — it's the official void-packages shell helper).
Binary installed as: `/usr/bin/vpm` (the actual binary we use day-to-day).
No conflict at runtime — the official `vpm` package is rarely installed alongside custom builds.
